# Testing with Zenstruck Foundry

## Overview

This project uses [Zenstruck Foundry](https://github.com/zenstruck/foundry) for test data management. Foundry provides a modern, fluent factory pattern for creating entities in tests and fixtures.

## Why Foundry?

### Advantages over LiipTestFixturesBundle

✅ **Cleaner API**: Fluent, expressive factory methods
✅ **No SQLite**: Uses database transactions directly
✅ **Type Safety**: Full IDE autocompletion
✅ **Flexibility**: Create entities inline or in fixtures
✅ **Modern**: Active development, Symfony 6+ support
✅ **Powerful**: Sequences, states, relationships

### Comparison

**Foundry:**
```php
UserFactory::new()
    ->withCredentials('john@example.com', 'password123')
    ->create();
```

**Old Approach:**
```php
$user = new User();
$user->setEmail('john@example.com');
$user->setPassword($this->passwordHasher->hashPassword($user, 'password123'));
$manager->persist($user);
$manager->flush();
```

## Creating Factories

### Basic Factory

```php
// src/Factory/UserFactory.php
namespace App\Factory;

use App\Entity\User;
use Zenstruck\Foundry\Persistence\PersistentProxyObjectFactory;

final class UserFactory extends PersistentProxyObjectFactory
{
    public static function class(): string
    {
        return User::class;
    }

    protected function defaults(): array
    {
        return [
            'email' => self::faker()->unique()->email(),
            'password' => 'password',
            'roles' => ['ROLE_USER'],
        ];
    }
}
```

### Factory with Custom Methods

```php
final class UserFactory extends PersistentProxyObjectFactory
{
    // ...existing code...

    public function withCredentials(string $email, string $password): self
    {
        return $this->with([
            'email' => $email,
            'password' => $password,
        ]);
    }

    public function admin(): self
    {
        return $this->with([
            'roles' => ['ROLE_USER', 'ROLE_ADMIN'],
        ]);
    }
}
```

### Factory with Initialization

```php
final class UserFactory extends PersistentProxyObjectFactory
{
    private static ?UserPasswordHasherInterface $passwordHasher = null;

    public static function setPasswordHasher(UserPasswordHasherInterface $hasher): void
    {
        self::$passwordHasher = $hasher;
    }

    protected function initialize(): static
    {
        return $this->afterInstantiate(function (User $user): void {
            if (self::$passwordHasher && !str_starts_with($user->getPassword(), '$')) {
                $hashed = self::$passwordHasher->hashPassword($user, $user->getPassword());
                $user->setPassword($hashed);
            }
        });
    }
}
```

## Using Factories in Tests

### Test Setup

```php
use ApiPlatform\Symfony\Bundle\Test\ApiTestCase;
use App\Factory\UserFactory;
use Symfony\Component\PasswordHasher\Hasher\UserPasswordHasherInterface;
use Zenstruck\Foundry\Test\ResetDatabase;
use Zenstruck\Foundry\Test\Factories;

class MyTest extends ApiTestCase
{
    use ResetDatabase; // Reset database before each test
    use Factories;     // Enable factory usage

    protected function setUp(): void
    {
        parent::setUp();
        
        // Configure factory
        $passwordHasher = static::getContainer()->get(UserPasswordHasherInterface::class);
        UserFactory::setPasswordHasher($passwordHasher);
    }
}
```

### Creating Entities

```php
// Single entity
$user = UserFactory::new()->create();

// With custom data
$user = UserFactory::new()
    ->withCredentials('john@example.com', 'password123')
    ->create();

// Multiple entities
UserFactory::createMany(5);

// Admin user
$admin = UserFactory::new()
    ->withCredentials('admin@example.com', 'admin123')
    ->admin()
    ->create();

// Without persisting
$user = UserFactory::new()->withoutPersisting()->create();

// Access the object
$email = $user->getEmail();
```

### Using Sequences

```php
// Create 10 users with sequential emails
UserFactory::new()
    ->sequence(fn($i) => ['email' => "user{$i}@example.com"])
    ->many(10)
    ->create();

// Create with different roles
UserFactory::new()
    ->sequence(function($i) {
        return $i % 2 === 0 
            ? ['roles' => ['ROLE_ADMIN']]
            : ['roles' => ['ROLE_USER']];
    })
    ->many(10)
    ->create();
```

### Complete Test Example

```php
public function testUserCanLogin(): void
{
    // Given: Create a test user
    UserFactory::new()
        ->withCredentials('john@example.com', 'password123')
        ->create();

    // When: User attempts to login
    $client = static::createClient();
    $response = $client->request('POST', '/token', [
        'headers' => ['Content-Type' => 'application/x-www-form-urlencoded'],
        'body' => http_build_query([
            'grant_type' => 'password',
            'client_id' => 'bank_app',
            'username' => 'john@example.com',
            'password' => 'password123',
            'scope' => 'email',
        ]),
    ]);

    // Then: User receives an access token
    $this->assertResponseIsSuccessful();
    $data = $response->toArray();
    $this->assertArrayHasKey('access_token', $data);
    
    // Database automatically reset before next test
}
```

## Using Factories in Fixtures

```php
// src/DataFixtures/UserFixtures.php
namespace App\DataFixtures;

use App\Factory\UserFactory;
use Doctrine\Bundle\FixturesBundle\Fixture;
use Doctrine\Persistence\ObjectManager;
use Symfony\Component\PasswordHasher\Hasher\UserPasswordHasherInterface;

class UserFixtures extends Fixture
{
    public function __construct(
        private readonly UserPasswordHasherInterface $passwordHasher
    ) {
    }

    public function load(ObjectManager $manager): void
    {
        // Configure factory
        UserFactory::setPasswordHasher($this->passwordHasher);

        // Create test users
        UserFactory::new()
            ->withCredentials('john@example.com', 'password123')
            ->create();

        UserFactory::new()
            ->withCredentials('jane@example.com', 'password456')
            ->create();

        UserFactory::new()
            ->withCredentials('admin@example.com', 'admin123')
            ->admin()
            ->create();
    }
}
```

## Database Reset Strategies

### Option 1: ResetDatabase (Recommended)

```php
use Zenstruck\Foundry\Test\ResetDatabase;

class MyTest extends ApiTestCase
{
    use ResetDatabase; // Resets before each test
    
    public function testSomething(): void
    {
        // Database is clean here
        UserFactory::createMany(5);
        // Will be reset before next test
    }
}
```

### Option 2: Per-Class Reset

```php
class MyTest extends ApiTestCase
{
    use ResetDatabase {
        resetDatabase as protected resetDatabaseTrait;
    }
    
    public static function setUpBeforeClass(): void
    {
        parent::setUpBeforeClass();
        static::resetDatabaseTrait(); // Reset once per class
    }
}
```

## Advanced Features

### States

Create factories with different states:

```php
final class UserFactory extends PersistentProxyObjectFactory
{
    public function active(): self
    {
        return $this->with(['status' => 'active']);
    }

    public function inactive(): self
    {
        return $this->with(['status' => 'inactive']);
    }

    public function verified(): self
    {
        return $this->with(['emailVerified' => true]);
    }
}

// Usage
UserFactory::new()->active()->verified()->create();
```

### Relationships

```php
// Create user with bank accounts
$user = UserFactory::new()->create();

BankAccountFactory::new()
    ->for($user) // Set the user relationship
    ->createMany(3);

// Or inline
BankAccountFactory::new()
    ->withAttributes([
        'user' => UserFactory::new(),
    ])
    ->create();
```

### Custom Faker

```php
protected function defaults(): array
{
    return [
        'email' => self::faker()->unique()->companyEmail(),
        'name' => self::faker()->name(),
        'bio' => self::faker()->paragraph(),
        'createdAt' => self::faker()->dateTimeBetween('-1 year'),
    ];
}
```

### Instantiate Without Persisting

```php
// Create object but don't persist
$user = UserFactory::new()->withoutPersisting()->create();

// Modify and then persist
$user->setEmail('new@example.com');
UserFactory::repository()->save($user);
```

## Best Practices

### ✅ Do's

1. **Use Factories for All Entities**
   ```php
   UserFactory::new()->create();  // ✅ Good
   new User(); $em->persist();    // ❌ Avoid
   ```

2. **Create Expressive Methods**
   ```php
   UserFactory::new()->admin()->verified()->create(); // ✅ Clear
   ```

3. **Use ResetDatabase Trait**
   ```php
   use ResetDatabase; // ✅ Automatic cleanup
   ```

4. **Configure Once in setUp**
   ```php
   protected function setUp(): void
   {
       parent::setUp();
       UserFactory::setPasswordHasher($this->passwordHasher);
   }
   ```

5. **Use Sequences for Bulk Creation**
   ```php
   UserFactory::new()->sequence(...)->many(100)->create();
   ```

### ❌ Don'ts

1. **Don't Manual Flush**
   ```php
   $factory->create(); // Already persisted and flushed
   ```

2. **Don't Mix Patterns**
   ```php
   // ❌ Don't mix factory and manual entity creation
   $user = UserFactory::new()->create();
   $account = new BankAccount(); // Use factory instead
   ```

3. **Don't Forget ResetDatabase**
   ```php
   // ❌ Missing trait = database pollution
   class MyTest extends ApiTestCase
   {
       // use ResetDatabase; // Don't forget!
   }
   ```

## Troubleshooting

### Issue: Factory not found
**Solution:** Run `composer dump-autoload`

### Issue: Database not reset
**Solution:** Ensure `use ResetDatabase;` trait is present

### Issue: Faker generates duplicates
**Solution:** Use `unique()` modifier
```php
'email' => self::faker()->unique()->email()
```

### Issue: Password not hashed
**Solution:** Set password hasher in factory initialization

## Performance Tips

1. **Use `createMany()` for bulk creation**
   ```php
   UserFactory::createMany(100); // Faster than loop
   ```

2. **Disable auto-refresh in tests**
   ```yaml
   # config/packages/test/zenstruck_foundry.yaml
   zenstruck_foundry:
       auto_refresh_proxies: false
   ```

3. **Use `withoutPersisting()` for mock data**
   ```php
   $user = UserFactory::new()->withoutPersisting()->create();
   ```

## Migration from LiipTestFixturesBundle

### Before (Liip)
```php
class MyTest extends ApiTestCase
{
    private $databaseTool;
    
    protected function setUp(): void
    {
        $this->databaseTool = static::getContainer()
            ->get(DatabaseToolCollection::class)
            ->get();
    }
    
    public function testSomething(): void
    {
        $this->databaseTool->loadFixtures([UserFixtures::class]);
        // test code
    }
}
```

### After (Foundry)
```php
use Zenstruck\Foundry\Test\ResetDatabase;
use Zenstruck\Foundry\Test\Factories;

class MyTest extends ApiTestCase
{
    use ResetDatabase;
    use Factories;
    
    public function testSomething(): void
    {
        UserFactory::createMany(3);
        // test code
    }
}
```
