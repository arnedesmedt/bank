# Cleanup and Makefile Symlinks - February 25, 2026

## Changes Made

### 🗑️ Removed Unnecessary Scripts

**Deleted:**
1. `backend/run-auth-tests.sh` - Redundant with `make test-backend`
2. `backend/load-fixtures.sh` - Redundant with `make fixtures-load`

**Reason:** These scripts duplicated functionality already available through the Makefile. Using a single interface (make) is cleaner and more maintainable.

### 🔗 Added Makefile Symlinks

**Created symlinks:**
1. `backend/Makefile` → `../Makefile`
2. `frontend/Makefile` → `../Makefile`

**Benefit:** You can now run make commands from any directory:

```bash
# From project root
make test-backend

# From backend directory
cd backend
make test-backend

# From frontend directory
cd frontend
make test-frontend
```

This improves developer experience by allowing commands to be run from the context you're working in.

### 📝 Updated Documentation

**Modified files:**
1. `QUICKSTART.md` - Added note about Makefile symlinks
2. `TEST_INFRASTRUCTURE_SUMMARY.md` - Removed script references
3. `FOUNDRY_MIGRATION.md` - Updated files list
4. `backend/tests/README.md` - Updated with make commands only

## Usage

### Running Tests

```bash
# Any of these work:
make test-backend                    # From root
cd backend && make test-backend      # From backend
docker compose exec php vendor/bin/phpunit  # Direct
```

### Loading Fixtures

```bash
# Any of these work:
make fixtures-load                   # From root
cd backend && make fixtures-load     # From backend
```

### All Available Commands

```bash
make help    # Show all commands
```

Common commands:
- `make up` - Start containers
- `make down` - Stop containers
- `make test-backend` - Run backend tests
- `make test-frontend` - Run frontend tests
- `make lint-backend` - Run backend linting
- `make fixtures-load` - Load development fixtures
- `make migrate` - Run database migrations
- `make shell-php` - Access PHP container
- `make shell-frontend` - Access frontend container
