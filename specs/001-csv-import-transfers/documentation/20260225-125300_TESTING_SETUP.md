# Testing Setup with Fixtures and Transactional Rollback

## Date
February 25, 2026

## Overview

The testing infrastructure follows [API Platform best practices](https://api-platform.com/docs/symfony/testing/) with:
- ✅ Doctrine fixtures for consistent test data
- ✅ Automatic database rollback after each test (via LiipTestFixturesBundle)
- ✅ Separate test database
- ✅ Fast test execution with database caching

## Architecture

### Test Infrastructure Components

1. **LiipTestFixturesBundle**
   - Provides `DatabaseTool` for loading fixtures
   - Uses SQLite for caching loaded fixtures
   - Automatic rollback after each test
   - Faster test execution

2. **DoctrineFixturesBundle**
   - Define reusable fixtures
   - Share fixtures between tests and dev environment
   - Organize test data

3. **API Platform Test Client**
   - Integration testing with real HTTP requests
   - Automatic Symfony kernel booting
   - Access to container services

## Fixtures

### User Fixtures (`src/DataFixtures/UserFixtures.php`)

Provides test users for authentication tests:

```php
class UserFixtures extends Fixture
{
    public function load(ObjectManager $manager): void
    {
        // Test user 1
        $user1 = new User();
        $user1->setEmail('john@example.com');
        $user1->setPassword($this->passwordHasher->hashPassword($user1, 'password123'));
        $manager->persist($user1);

        // Test user 2
        $user2 = new User();
        $user2->setEmail('jane@example.com');
        $user2->setPassword($this->passwordHasher->hashPassword($user2, 'password456'));
        $manager->persist($user2);

        // Admin user
        $admin = new User();
        $admin->setEmail('admin@example.com');
        $admin->setPassword($this->passwordHasher->hashPassword($admin, 'admin123'));
        $admin->setRoles(['ROLE_USER', 'ROLE_ADMIN']);
        $manager->persist($admin);

        $manager->flush();
    }
}
```

**Test Users:**
- `john@example.com` / `password123` - Regular user
- `jane@example.com` / `password456` - Regular user
- `admin@example.com` / `admin123` - Admin user

## Test Pattern

### Standard Test Structure

```php
use ApiPlatform\Symfony\Bundle\Test\ApiTestCase;
use Liip\TestFixturesBundle\Services\DatabaseToolCollection;
use Liip\TestFixturesBundle\Services\DatabaseTools\AbstractDatabaseTool;

class AuthenticationTest extends ApiTestCase
{
    private ?AbstractDatabaseTool $databaseTool = null;

    protected function setUp(): void
    {
        parent::setUp();
        $this->databaseTool = static::getContainer()
            ->get(DatabaseToolCollection::class)
            ->get();
    }

    public function testSomething(): void
    {
        // Load fixtures - database is automatically reset before
        $this->databaseTool->loadFixtures([UserFixtures::class]);

        // Make requests
        $client = static::createClient();
        $response = $client->request('GET', '/api/me');

        // Assertions
        $this->assertResponseIsSuccessful();
        
        // Database automatically rolled back after test
    }
}
```

### Key Features

1. **Automatic Database Reset**: `loadFixtures()` resets database before loading
2. **Transactional Rollback**: Changes are rolled back after each test
3. **Fast Execution**: SQLite cache makes repeated fixture loading fast
4. **Isolation**: Each test runs with clean state

## Creating New Fixtures

### 1. Create Fixture Class

```php
// src/DataFixtures/BankAccountFixtures.php
namespace App\DataFixtures;

use Doctrine\Bundle\FixturesBundle\Fixture;
use Doctrine\Persistence\ObjectManager;
use Doctrine\Common\DataFixtures\DependentFixtureInterface;

class BankAccountFixtures extends Fixture implements DependentFixtureInterface
{
    public function load(ObjectManager $manager): void
    {
        $bankAccount = new BankAccount();
        $bankAccount->setAccountName('Main Account');
        $bankAccount->setAccountNumber('BE12345678901234');
        $manager->persist($bankAccount);
        
        $manager->flush();
    }
    
    public function getDependencies(): array
    {
        return [UserFixtures::class]; // Load UserFixtures first
    }
}
```

### 2. Use in Tests

```php
public function testWithBankAccounts(): void
{
    $this->databaseTool->loadFixtures([
        UserFixtures::class,
        BankAccountFixtures::class,
    ]);
    
    // Test code...
}
```

## Fixture Dependencies

Order fixtures with `DependentFixtureInterface`:

```php
class TransferFixtures extends Fixture implements DependentFixtureInterface
{
    public function getDependencies(): array
    {
        return [
            UserFixtures::class,
            BankAccountFixtures::class,
        ];
    }
}
```

LiipTestFixturesBundle will load them in correct order.

## Performance Tips

### 1. Cache Fixtures
LiipTestFixturesBundle caches loaded fixtures in SQLite. First load is slow, subsequent loads are fast.

### 2. Load Only Required Fixtures
```php
// Don't load all fixtures if you only need users
$this->databaseTool->loadFixtures([UserFixtures::class]);
```

### 3. Shared Fixtures for Test Class
```php
class MyTest extends ApiTestCase
{
    protected function setUp(): void
    {
        parent::setUp();
        $this->databaseTool = static::getContainer()
            ->get(DatabaseToolCollection::class)
            ->get();
        
        // Load once for all tests in this class
        $this->databaseTool->loadFixtures([UserFixtures::class]);
    }
}
```

## Troubleshooting

### Issue: "Class 'DatabaseToolCollection' not found"
**Solution:** Install liip/test-fixtures-bundle
```bash
docker compose exec php composer require --dev liip/test-fixtures-bundle
```

### Issue: "Database does not exist"
**Solution:** Create test database
```bash
docker compose exec php bin/console doctrine:database:create --env=test
docker compose exec php bin/console doctrine:migrations:migrate --env=test --no-interaction
```

### Issue: "Fixtures fail to load"
**Solution:** Check fixture dependencies and entity definitions

### Issue: "Tests are slow"
**Solution:** 
1. Ensure LiipTestFixturesBundle is configured
2. Check SQLite cache is working
3. Load only required fixtures

## Best Practices

### ✅ Do's
- Use fixtures for consistent test data
- Load only required fixtures per test
- Use descriptive fixture names
- Add fixture dependencies when needed
- Keep fixtures simple and focused
- Use same fixtures for tests and dev

### ❌ Don'ts
- Don't manually create entities in tests (use fixtures)
- Don't forget to load fixtures in setUp()
- Don't share mutable state between tests
- Don't load unnecessary fixtures
- Don't hardcode UUIDs in fixtures

## Example: Complete Test

```php
<?php

namespace App\Tests\Api;

use ApiPlatform\Symfony\Bundle\Test\ApiTestCase;use App\Tests\DataFixtures\UserFixtures;use Liip\TestFixturesBundle\Services\DatabaseToolCollection;

class ExampleTest extends ApiTestCase
{
    private $databaseTool;

    protected function setUp(): void
    {
        parent::setUp();
        $this->databaseTool = static::getContainer()
            ->get(DatabaseToolCollection::class)
            ->get();
    }

    public function testCompleteFlow(): void
    {
        // Given: Users exist in database
        $this->databaseTool->loadFixtures([UserFixtures::class]);

        $client = static::createClient();

        // When: User logs in
        $tokenResponse = $client->request('POST', '/token', [
            'headers' => ['Content-Type' => 'application/x-www-form-urlencoded'],
            'body' => http_build_query([
                'grant_type' => 'password',
                'client_id' => 'bank_app',
                'username' => 'john@example.com',
                'password' => 'password123',
                'scope' => 'email',
            ]),
        ]);

        $this->assertResponseIsSuccessful();
        $token = $tokenResponse->toArray()['access_token'];

        // When: User accesses protected resource
        $response = $client->request('GET', '/api/me', [
            'headers' => [
                'Authorization' => 'Bearer ' . $token,
                'Content-Type' => 'application/json',
            ],
        ]);

        // Then: User data is returned
        $this->assertResponseIsSuccessful();
        $userData = $response->toArray();
        $this->assertSame('john@example.com', $userData['email']);

        // Database automatically rolled back after test!
    }
}
```
