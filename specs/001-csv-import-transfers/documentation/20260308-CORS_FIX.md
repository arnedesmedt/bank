# CORS Configuration Fix

**Date:** March 8, 2026  
**Issue:** Frontend on `http://0.0.0.0:3000` getting CORS errors when accessing backend API on `http://localhost:8080`

## Problem

When the frontend (running on `0.0.0.0:3000`) tries to make requests to the backend API (running on `localhost:8080`), browsers block the requests due to Cross-Origin Resource Sharing (CORS) policy.

### CORS Error Messages

```
Access to fetch at 'http://localhost:8080/token' from origin 'http://0.0.0.0:3000' 
has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present 
on the requested resource.
```

## Solution Implemented

### 1. Installed NelmioCorsBundle

```bash
docker compose exec php composer require nelmio/cors-bundle
```

### 2. Configured CORS Headers

**File:** `backend/config/packages/nelmio_cors.yaml`

```yaml
nelmio_cors:
    defaults:
        origin_regex: true
        allow_origin: ['%env(CORS_ALLOW_ORIGIN)%']
        allow_methods: ['GET', 'OPTIONS', 'POST', 'PUT', 'PATCH', 'DELETE']
        allow_headers: ['Content-Type', 'Authorization', 'Accept', 'X-Requested-With']
        expose_headers: ['Link', 'Content-Disposition']
        max_age: 3600
        allow_credentials: true
    paths:
        '^/': 
            allow_origin: ['*']
            allow_headers: ['*']
            allow_methods: ['POST', 'PUT', 'PATCH', 'GET', 'DELETE', 'OPTIONS']
            max_age: 3600
```

**Configuration Explained:**
- `allow_origin: ['*']` - Allows requests from any origin (development mode)
- `allow_headers: ['*']` - Allows all headers
- `allow_methods` - Allows all HTTP methods needed for REST API
- `allow_credentials: true` - Allows cookies/auth headers
- `max_age: 3600` - Preflight cache duration (1 hour)

### 3. Added Environment Variables

**Files:** `backend/.env` and `backend/.env.local`

```dotenv
###> nelmio/cors-bundle ###
CORS_ALLOW_ORIGIN='^https?://(localhost|127\.0\.0\.1|0\.0\.0\.0)(:[0-9]+)?$'
###< nelmio/cors-bundle ###
```

This regex pattern allows:
- `http://localhost` and `http://localhost:*`
- `http://127.0.0.1` and `http://127.0.0.1:*`
- `http://0.0.0.0` and `http://0.0.0.0:*`
- `https://` versions of all above

### 4. Restarted Services

```bash
docker compose restart php nginx
```

## How CORS Works

### Preflight Request (OPTIONS)

For "complex" requests (POST with custom headers), browsers send an OPTIONS request first:

```
OPTIONS /token HTTP/1.1
Origin: http://0.0.0.0:3000
Access-Control-Request-Method: POST
Access-Control-Request-Headers: content-type
```

Server responds with allowed methods and headers:

```
Access-Control-Allow-Origin: http://0.0.0.0:3000
Access-Control-Allow-Methods: POST, GET, PUT, DELETE
Access-Control-Allow-Headers: content-type, authorization
Access-Control-Max-Age: 3600
```

### Actual Request

If preflight succeeds, browser sends the actual request:

```
POST /token HTTP/1.1
Origin: http://0.0.0.0:3000
Content-Type: application/x-www-form-urlencoded
```

Server must include CORS headers in response:

```
Access-Control-Allow-Origin: http://0.0.0.0:3000
```

## Production Considerations

### ⚠️ Current Configuration is Development-Only

The current CORS configuration allows requests from **any origin** (`allow_origin: ['*']`). This is **NOT secure for production**.

### For Production:

1. **Restrict allowed origins** to specific domains:

```yaml
nelmio_cors:
    paths:
        '^/':
            allow_origin: ['https://yourdomain.com']
            allow_credentials: true
```

2. **Use environment-specific configuration:**

```yaml
# config/packages/prod/nelmio_cors.yaml
nelmio_cors:
    paths:
        '^/':
            allow_origin: ['%env(PRODUCTION_FRONTEND_URL)%']
```

3. **Set in production .env:**

```dotenv
PRODUCTION_FRONTEND_URL=https://app.yourdomain.com
```

## Troubleshooting

### CORS Headers Not Appearing

1. **Clear Symfony cache:**
   ```bash
   docker compose exec php bin/console cache:clear
   ```

2. **Restart containers:**
   ```bash
   docker compose restart php nginx
   ```

3. **Check bundle is registered:**
   ```bash
   grep NelmioCorsBundle backend/config/bundles.php
   ```

### Still Getting CORS Errors

1. **Check browser console** for actual error message
2. **Verify Origin header** matches allowed origins
3. **Check preflight request** in Network tab (OPTIONS request)
4. **Ensure HTTPS/HTTP match** between frontend and backend
