# Implementation Plan: Refund Linking Improvements

**Branch**: `010-refund-linking-improvements` | **Date**: 2026-03-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/010-refund-linking-improvements/spec.md`

## Summary

Implement improved refund transfer linking and parent/child logic for bank transfers. Users can select a transfer and link refund transfers as children, with the parent amount reduced by the sum of refunds. UI groups refunds under parents (accordion). A new `amount_before_refund` column preserves the original amount. All changes are backward compatible and handle edge cases (orphaned refunds, excessive refund amounts, duplicate linking).
